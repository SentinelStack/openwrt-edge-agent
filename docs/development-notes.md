# Development Notes

## Why Docker is used
Docker is used only for build and cross-compilation on macOS. The final binary runs directly on the OpenWrt router, not inside Docker.

## Why `--platform linux/amd64` is required on Apple Silicon
The OpenWrt 23.05.3 toolchain used by this project is distributed for Linux x86_64. On Apple Silicon hosts, Docker must run the container with `--platform linux/amd64` so the toolchain can execute correctly.

## Why we use `scp -O`
OpenWrt images often do not include `/usr/libexec/sftp-server` by default. Modern `scp` can try SFTP mode first, which fails in that case. `scp -O` forces legacy SCP protocol and avoids this deployment issue.

## Why compiling directly on router is not recommended
Minimal OpenWrt systems may not have a full compiler toolchain installed. Even when available, versions and libraries may differ from what is needed. Cross-compiling in Docker keeps builds reproducible and consistent.

## Why traffic generated from Mac may not appear
The sensor only sees traffic that passes through the router interface or is generated by the router itself. If the Mac is not using the OpenWrt router as gateway (for example direct hotspot path), traffic may not be visible to the raw socket filter.

## Useful router commands

```sh
ssh root@192.168.1.1
opkg update

tcpdump -D
/root/net_filter tcp

```
